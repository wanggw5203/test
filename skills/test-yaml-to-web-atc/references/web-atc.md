# Web ATC 生成规则

## 必需输入

生成前尽可能收集：

- 功能 YAML：确定验证目标。
- TAD、TRD 或原始需求：确认业务规则。
- 已成功运行的 `.atc.yaml` 黄金脚本：确认页面操作方法。
- 用户确认的范围：例如 `仅保存` 或 `保存并提交`。
- 真实页面证据：截图、`relationPages`、Axure 或浏览器观察结果。
- 上传资源及其真实路径。
- 项目的 `package.json`、`config.yaml` 和现有 ATC 示例。

没有黄金脚本或真实页面证据时，把输出标记为 `待真实页面校准`。

## 常用结构

优先沿用附近 `.atc.yaml`，常见结构如下：

```yaml
suite:
  name: 场景名称
  platform: dhr
TestCases:
  - name: 添加待入职人员-完整信息并保存
    description: 填写核心信息、上传附件并保存成功
    path: /third-employment/food-manage/onboarding/add-staff
    functionalCaseIds:
      - functional-case-id
    relationPages:
      - https://example.invalid/reference.png
    timeout: 600000ms
    variables: {}
    before: []
    steps:
      - step: 填写姓名
        action: input
        target: 「基础信息」区域的「姓名」输入框
        data:
          value: ${personName}
        expected: 输入框显示本次生成的姓名
```

常用动作：

- `input`：输入值。
- `select`：选择选项。
- `click`：点击控件。
- `upload`：上传真实存在的文件。
- `scroll`：滚动到非首屏区域。
- `assert`：验证一个页面状态。
- `wait`：等待已知页面变化。
- `navigate`：打开有证据支持的页面或路由。
- `ai`：仅表达一个无法用确定性动作稳定完成的复杂交互。

## 信息优先级

按信息类型处理冲突：

- 自动化范围：用户确认优先。
- 业务规则：已确认需求或 TAD 优先，其次是功能 YAML。
- 页面操作顺序：黄金 ATC 优先，其次是真实页面证据，再次是 Axure。
- 字段标签、占位符、提示语和跳转：真实页面证据优先，不得虚构。
- 上传文件：真实存在的资源优先，不得引用缺失文件。

把未解决冲突记录在结果说明或用例注释中。

## 默认生成策略

第一版默认生成：

- 一条稳定 P0 主流程。
- 用户未确认提交或审批时仅覆盖保存。
- 不生成注释掉的批量用例。
- 不保留占位值。
- 不引用缺失的上传文件。

用户要求扩展时再生成：

- P1 完整字段成功路径。
- P1/P2 必填、唯一性、边界、条件展示、上传失败和权限场景。
- 不同业务断言分别生成用例。

## 数据规则

- 成功路径对唯一字段使用合法、动态、唯一数据。
- 中文姓名字段不得追加数字。
- 可重复成功路径不得使用固定手机号或身份证号。
- 页面校验身份证校验位时使用合法身份证号码。
- 证件或合同失效日期使用相对日期或未来安全日期。
- 紧急联系人信息与被创建人员不同。
- 固定重复数据只用于重复校验用例。

## 交互规则

- 一个步骤执行一个动作或断言。
- 上传、裁剪确认、滚动、保存和最终断言分别成步。
- 操作非首屏控件前增加滚动步骤。
- 复杂组件沿用黄金脚本的树选择、级联、裁剪、等待和右键处理。
- 只使用证据支持的字段标签和页面结构。
- 真实页面占位符为 `请输入` 时，不得虚构为 `请输入姓名`。
- 不得把“上传多个附件、保存、提交并断言成功”合并为一个 `ai` 步骤。
- 保存后页面关闭时，断言成功提示或返回列表记录。

## 校验清单

项目执行器可用时，从项目根目录运行其校验命令，例如 `npm run case:validate`。

执行器不可用时检查：

- `suite.name` 和 `platform` 存在。
- 每条用例包含 `name`、`description`、`path`、`functionalCaseIds`、`timeout` 和 `steps`。
- 每个上传文件存在于项目资源或资产搜索路径中。
- 每个步骤包含 `step`、`action` 和 `target`。
- `data.value` 和 `data.file` 引用已声明变量或真实文件。
- 没有 `ai` 步骤包含多个操作或多个场景。
- 最终断言与用户确认的自动化范围一致。
