# ATC 失败与修复规则

## 根因分类

| 分类 | 典型证据 | 主处理目标 |
| --- | --- | --- |
| `ATC_PARSE_OR_SCHEMA_ERROR` | YAML 解析失败、字段缺失、动作不受支持、变量未声明 | ATC 文件 |
| `LOCATOR_OR_PAGE_DRIFT` | 控件标签、层级、组件或页面结构与黄金脚本不同 | ATC 与页面证据 |
| `ACTION_SEQUENCE_OR_WAIT_ERROR` | 未滚动、弹窗未确认、保存前状态未完成、等待条件错误 | ATC 步骤顺序 |
| `TEST_DATA_OR_PREREQUISITE_ERROR` | 数据重复、过期、格式非法、前置状态不存在、资源缺失 | 变量、造数、资源 |
| `AUTH_OR_PERMISSION_ERROR` | 登录失效、403、角色无入口、数据范围不足 | 用户配置与权限 |
| `ENV_OR_SERVICE_ERROR` | DNS、超时、502/503、页面资源失败、环境配置错误 | 环境与服务 |
| `ASSERTION_OR_EXPECTATION_ERROR` | 被测动作完成，但观察结果与有依据的期望不同 | 证据、ATC 断言 |
| `PRODUCT_BEHAVIOR_ERROR` | 有效操作稳定复现违反已确认业务规则的行为 | 产品缺陷证据 |
| `REQUIREMENT_OR_CASE_CONFLICT` | TAD、功能 YAML、ATC 或真实页面互相矛盾 | 上游规则确认 |
| `UNKNOWN` | 当前证据不能支持单一根因 | 补充日志与页面证据 |

## 诊断顺序

1. 先确认 YAML 是否解析、资源是否存在。
2. 再确认环境、登录、权限和页面入口是否可用。
3. 确认前置数据和动态变量是否合法。
4. 确认失败步骤前的动作是否全部完成。
5. 对比黄金脚本、真实页面和失败截图中的控件与状态。
6. 只有业务动作确实完成后，才诊断最终断言。

## 可自动修复

- YAML 缩进、字段、动作类型和变量引用存在确定性错误。
- 上传资源路径与真实文件不一致。
- 黄金脚本或真实页面能证明正确控件标签与操作顺序。
- 缺少已验证的滚动、裁剪确认、弹窗确认或状态等待。
- 成功路径使用了固定、重复、过期或格式非法的数据。
- 项目配置能证明执行命令或 Profile 选择错误。

## 禁止自动修复

- 因实际行为不同而直接修改业务期望。
- 删除失败步骤、弱化断言或把用例改为跳过。
- 用固定等待掩盖确定性顺序错误。
- 对非幂等创建操作进行无上限重试。
- 用另一条历史静态数据替换失效数据。
- 缺少真实页面证据时猜测字段、路由、提示语或组件行为。

## 重试边界

只有操作幂等、失败具有瞬时证据、次数有上限且保留首轮日志时才允许重试。解析错误、权限拒绝、确定性数据错误、断言冲突和非幂等创建操作不得盲目重试。

## 上游回写矩阵

| 根因 | 回写目标 | 处理规则 |
| --- | --- | --- |
| ATC 结构、定位、动作、等待 | `.atc.yaml` | 依据黄金脚本或真实页面最小修改 |
| 动态数据或资源 | ATC 变量、资源、造数规则 | 保持功能断言不变 |
| 功能 YAML 漏场景 | 功能 YAML | 增加用例并追溯 TAD 规则 |
| 期望与已确认规则冲突 | 功能 YAML、ATC | 先获得业务证据，再同步修改 |
| TAD 规则缺失或矛盾 | TAD -> YAML -> ATC | 按上游到下游顺序传播 |
| 环境、权限、执行器 | 配置、预检、运行规则 | 不修改业务期望 |
| 产品缺陷 | 缺陷证据、失败报告 | 保留失败用例，不吸收错误行为 |
