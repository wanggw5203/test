# 记录创建需求

## 背景

运营人员需要通过记录管理服务创建业务记录，后续审核流程会使用新记录的标识。

## 业务规则

1. 管理员可以创建标准记录，审核员只能查看记录。
2. 创建记录时必须提供记录类型、所属上级记录和唯一名称。
3. 记录类型包含“标准”和“临时”。
4. 上级记录必须存在且处于有效状态。
5. 名称在同一上级记录下不能重复。
6. 创建成功返回记录 ID 和成功状态。
7. 缺少必填字段、使用无效上级记录或名称重复时应返回明确业务错误，不得创建数据。

## 接口线索

- 动作：创建记录
- 请求：上级记录 ID、角色编码、记录类型、名称
- 响应：成功状态、记录 ID 或业务错误
- 需要覆盖：成功、权限不足、必填缺失、无效上级记录、名称重复
